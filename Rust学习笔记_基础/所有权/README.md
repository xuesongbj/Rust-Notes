# 所有权

一个值的所有权被转移给另外一个变量的绑定过程，就叫做所有权转移。

Rust中每一个值都有一个所有者，也就是说Rust中分配的每块内存都有其所有者，**所有者负责该内存的释放和读写权限**，并且每次每个值只能有唯一的所有者。这就是Rust的所有权机制(OwnerShip)。

## 所有权机制(OwnerShip)

Rust中，可以通过是否实现`Copy` trait进行区分数据类型是**值语义**还是**引用语义**。Rust也引入了新的语义：**复制(Copy)语义** 和 **移动(Move)语义**。

**复制语义**对应值语义来理解，也就是说，实现了 `Copy` 的类型在进行按位复制时是安全的；**移动语义**可以对应引用语义来理解，也就是说，在其他语言(C++)中本来是引用语义的类型，在Rust中不允许按位复制，只允许移动所有权，只有这样才能保证安全。这样划分是因为引入了所有权机制，在所有权机制下同时保证内存安全和性能。
